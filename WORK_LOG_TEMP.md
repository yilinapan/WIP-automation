## 專案概述
WIP 轉貼紙工具是一個以 Streamlit 為介面的內部小工具，讓使用者上傳 WIP Excel，經由 Python 自動清洗與重組資料，輸出符合貼紙印製規格的 Excel，並在同一頁面中自動完成結果驗證。

---

## 開發歷程（依 agent 對話整理）

### 1. 建立初版 Streamlit 面板與處理流程
- 設計 Streamlit 介面：上傳 WIP Excel、選擇要處理的工作表、輸入／預設輸出檔名、「轉換成貼紙」按鈕、處理中提示與完成後下載按鈕。
- 實作 `processor.py`：從指定工作表讀取資料、略過前兩列標題、從第 3 列開始依欄位名稱解析資料。
- 完成輸出 Excel：依規格輸出 `MPO#、Style Number、中文名稱、size、COLOR、SKU、Price (NT)、吊牌貼紙數量(PCS)、單件袋貼紙數量(PCS)`，並提供下載功能與預覽區塊。
- 建立 `requirements.txt`、`install.bat` 與 `run.bat`，讓 Windows 使用者可以雙擊安裝與啟動。

### 2. Excel 結構理解與格式／顏色設定
- 解析 WIP 檔案實際結構：STYLE、中文品名、售價等欄位為合併儲存格，需要向下填滿；Color code、顏色欄位需清洗成符合輸出格式。
- 根據截圖整理 Excel 內建色彩（佈景主題色、標準色、ColorIndex 56 色），建立 `Excel內建顏色對照表.md`，方便之後用 openpyxl 設定背景色。
- 套用格式需求：標題列底色（淡黃色）、A 欄紅字與底色、不同 MPO 區塊交錯底色、欄寬與列高設定、文字對齊與金額千分位顯示。

### 3. 貼紙數量與展開邏輯修正
- 釐清貼紙數量計算：每個尺寸 S/M/L/XL 的「吊牌貼紙數量」與「單件袋貼紙數量」皆為「大貨交期數量 + 樣品交期數量 + 5」。
- 根據實際 WIP 結構，處理一款顏色可能有多個「大貨交期」與「樣品交期」列，改為依 (Style, Color code) 分組，累加所有交期列的數量後再加常數 5。
- 避免重複 Style：同一個 Style+Color 在多個區塊或小計列出現時，只保留一組完整 S/M/L/XL 資料。

### 4. 動態標題列、多區塊與小計列處理
- 改為動態搜尋標題列：以第 3 列為模板，在整張工作表中找到所有結構相同的標題列，將每段標題以下到下一個標題列前視為一個資料區塊。
- 僅對合併欄位做 forward fill：例如 STYLE、中文品名、售價、Color code、顏色等；避免把原本空白的加總列誤填成有效資料。
- 透過「階段」欄位內容（包含「大貨交期」「樣品交期」）過濾：只保留交期列，排除每款最後的數字加總列，避免多出一組大數字紀錄。

### 5. 驗證工具設計與整合（方案一～三）
- 撰寫《驗證工具方案建議》，規劃三種層級：
  - 方案一：來源與輸出數量加總比對。
  - 方案二：輸出檔格式與結構檢查。
  - 方案三：來源與輸出雙向比對（最完整）。
- 實作並整合驗證邏輯到同一個 Streamlit 頁面（不另開分頁、不要再上傳檔案）：
  - 使用來源 WIP 與剛產生的輸出結果自動執行驗證。
  - 方案一：逐筆比對每個 Style+Color+Size 的貼紙數量是否與來源計算結果一致。
  - 方案二：檢查每組 Style+Color 是否有完整 S/M/L/XL 四筆、Style Number 格式正確、COLOR 是否為純中文、數量為正整數、售價有值。
  - 方案三：補上來源→輸出與輸出→來源的雙向檢查，偵測是否有遺漏款式、額外多出的款式，以及中文品名／售價／顏色是否與來源完全一致。
- 驗證結果顯示方式：在同一頁面於轉換完成後自動出現「驗證結果」區塊，以 ✅ / ⚠️ / ❌ 與可展開清單呈現各類檢查狀態。

### 6. Streamlit UX 改善與狀態管理
- 調整工作表選擇 UI：處理工作表名稱顯示寬度，避免標籤截斷，並簡化副標標示（僅顯示偵測到的工作表數量）。
- 處理隱藏工作表：利用 openpyxl 檢查 sheet_state，區分可見與隱藏工作表，避免使用者誤解。
- 修正檔案清除行為：當使用者在上傳元件中按 X 清除檔案時，自動清空 session state 中的輸出檔 bytes 與預覽 DataFrame，讓畫面回到「尚未處理」狀態，不需手動重新整理。

### 7. 跨平台與 Mac 使用方案探討
- 設計 Windows 端的一鍵安裝與啟動流程：`install.bat` 自動建立虛擬環境並安裝套件，`run.bat` 一鍵啟動面板。
- 討論 Mac 端需求：希望同事不需使用終端機即可啟動工具。
- 提出 macOS 兩層方案：
  - 方案 A：`run_mac.command` 腳本，首次執行自動建立虛擬環境與安裝套件，之後雙擊即啟動 Streamlit。
  - 方案 B：使用 Automator 打包成 `.app`，將上述腳本內嵌，讓同事僅需雙擊 App 即可使用。
- 撰寫 `Mac使用說明_Automator與腳本.md`，記錄未來在 Mac 上實作腳本與 Automator `.app` 的完整步驟，預留之後擴充為真正打包獨立 App 的空間。

## 技術棧

- **語言與環境**
  - Python 3
  - Windows（主要開發環境）、macOS 使用情境規劃

- **後端邏輯與資料處理**
  - `pandas`：讀取與清洗 Excel、分區塊處理、多列合計與展開 S/M/L/XL。
  - `openpyxl`：控制 Excel 格式、字型、儲存格顏色、欄寬列高與合併儲存格。
  - 自訂資料處理函式：解析合併儲存格、動態尋找標題列、分組加總（按 Style+Color+Size）、字串清洗（純中文顏色）、欄位比對與驗證。

- **前端與介面**
  - Streamlit：上傳檔案、表單輸入、自訂按鈕與 loading 狀態、表格預覽、結果訊息區塊。
  - 基本 CSS 調整：修正 multiselect 標籤寬度等細節。

- **自動化與啟動腳本**
  - Windows：`install.bat`、`run.bat`。
  - macOS（設計中）：`run_mac.command` 腳本與 Automator `.app` 概念。

- **文件與維運**
  - 專案文件：`Excel內建顏色對照表.md`、`驗證工具方案建議.md`、`Mac使用說明_Automator與腳本.md`。
  - Agent transcripts 驅動的工作日誌整理流程。
